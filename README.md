## System Architecture

The Portfolio Publisher system is designed for efficient generation and management of listing artifacts. The core components are:

*   **`apps/`**: Stores structured JSON definitions for each of the 25 SaaS applications. These files serve as the single source of truth for all listing data.
*   **`prompts/`**: Contains templates used for generating various platform-specific or artifact-specific content. This allows for consistent and scalable content generation.
*   **`output/`**: Serves as the destination for all generated artifacts, including the formatted Flippa listings in Markdown, and potentially other output formats.
*   **`logs/`**: Stores JSONL formatted logs for tracking generation runs, errors, and other operational data.
*   **`src/`**: Houses the main application logic, divided into several sub-directories:
    *   **`config/`**: Manages configuration settings for Language Model (LLM) integrations and environment-specific variables.
    *   **`core/`**: Provides shared utilities, such as data loaders and prompt rendering engines, used across different parts of the system.
    *   **`publisher/`**: Orchestrates the generation process, taking app definitions and prompt templates to produce desired outputs.
    *   **`ui/`**: Contains components and logic for a potential command center user interface, enabling interactive management of the publishing workflow.
    *   **`api/`**: Defines HTTP endpoints for interacting with the system programmatically, particularly if integrated into a web framework like Next.js or Express.
